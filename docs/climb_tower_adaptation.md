# 爬塔功能其他語系適配指南

本指南說明如何為非台服（繁體中文）的語系版本適配爬塔功能。
台服適配由 `assets/resource/tw/` 提供，可作為最完整的參考範例。

---

## 架構概觀

```
assets/resource/
├── base/                   ← 官服（簡體中文）原始資源，不要直接修改
│   └── pipeline/climb_tower/
└── tw/                     ← 台服覆蓋層，只寫「與 base 不同」的節點
    ├── pipeline/climb_tower/
    │   └── climb_tower_tw.json   ← OCR 文字節點覆蓋
    └── image/              ← 若圖片模板不同，放這裡

agent/
├── custom/action/
│   └── tower_loop.py       ← Python 狀態機（主邏輯，通常不需動）
├── presets/
│   └── *.json              ← 使用者 Buff 優先級作業檔案
└── main_refactor.py        ← Agent 入口

assets/
└── interface.json          ← UI 任務選項定義
```

MaaFramework 會將語系覆蓋層**合併覆蓋** base，只需要寫有差異的節點。

---

## Step 1：建立語系覆蓋目錄

以日服（`jp`）為例：

```
assets/resource/jp/pipeline/climb_tower/climb_tower_jp.json
```

`interface.json` 裡需要對應的 `resource` 設定指向 `jp`，參考現有的 `tw` 設定。

---

## Step 2：OCR 節點翻譯

以下是所有需要翻譯的 OCR 節點及其對應的辨識文字（台服版本）。
找到你版本的對應文字後，在你的覆蓋 JSON 裡逐一覆蓋。

### 必須翻譯的節點

| 節點名稱 | 台服文字 | 說明 |
|---|---|---|
| `塔_偵測_完成` | `探索完成`、`當前房間`、`收集潛能` | 塔探索結束畫面 |
| `塔_偵測_等級提升` | `等級提升` | 升級動畫標題 |
| `塔_偵測_保存紀錄` | `保存紀錄` | 右下角存檔按鈕 |
| `塔_偵測_商店節點` | `商店購物` | 樓層選擇畫面的商店入口文字 |
| `塔_偵測_強化可用` | `強化`、`免費`、`免费` | 樓層選擇畫面的獨立強化按鈕 |
| `塔_偵測_上樓` | `上樓` | 前往下一層按鈕 |
| `塔_偵測_最終離開` | `離開星塔` | 最後一層的離開按鈕 |
| `塔_偵測_點選空白` | `點選空白`、`處繼續`、`處關閉` | 取得 Buff 後的提示文字 |
| `塔_偵測_默契提升` | `默契`、`提升` | 默契提升畫面 |
| `塔_OCR_拿走` | `拿走` | Buff 選擇後的確認按鈕 |
| `塔_商店_售罄` | `售罄` | 商店格子已售出標記 |
| `塔_商店_錢不夠` | `沒有足夠的` | 購買失敗彈窗 |
| `塔_商店_購買按鈕` | `購買` | 商店詳情面板的購買按鈕 |
| `塔_商店_優惠` | `優惠` | 商店格子折扣標籤 |
| `塔_商店_buff類型` | `可獲得新潛能` | 商品類型判斷：Buff |
| `塔_商店_音符類型` | `的音符` | 商品類型判斷：音符 |
| `塔_商店_音符激活` | `協奏`、`相關`、`技能` | 音符是否與隊伍相關（購買依據） |
| `塔_商店_購買確認` | `購買` | 購買二次確認彈窗 |
| `塔_商店_無法重置` | `無法` | 商店無法重置彈窗 |
| `塔_偵測_離開確認` | `是否離開星塔`、`輝光幣`、`目前尚有` | 離塔確認對話框 |
| `塔_偵測_確認按鈕` | `確認`、`确认`、`確.?認` | 對話框確認按鈕（regex） |

### 對話選項（需進遊戲確認）

`塔_偵測_對話選項` 節點包含約 20 個劇情對話選項文字，這些文字**因語系差異最大**，
建議實際進塔遊玩後對照截圖補齊。

```json
"塔_偵測_對話選項": {
    "recognition": {
        "type": "OCR",
        "param": {
            "expected": [
                "選項1文字", "選項2文字", "..."
            ],
            "roi": [350, 151, 870, 475]
        }
    },
    "action": {"type": "DoNothing"},
    "timeout": 1000
}
```

## Step 3：覆蓋 JSON 範例

只需要寫「有差異的節點」，完整範例參考 `tw/pipeline/climb_tower/climb_tower_tw.json`。

```json
{
    "塔_偵測_完成": {
        "recognition": {
            "type": "OCR",
            "param": {
                "expected": ["YOUR_LANG_TEXT_1", "YOUR_LANG_TEXT_2"],
                "roi": [449, 219, 399, 302]
            }
        },
        "action": {"type": "DoNothing"}
    },

    "塔_偵測_上樓": {
        "recognition": {
            "type": "OCR",
            "param": {
                "expected": ["YOUR_GO_UP_TEXT"],
                "roi": [421, 487, 287, 59]
            }
        },
        "action": {"type": "DoNothing"}
    }
}
```

> **ROI 格式**：`[x, y, width, height]`，座標以 1280×720 為基準。
> 若畫面解析度不同，ROI 可能需要微調。

## Step 4：圖片模板確認

以下節點使用圖片比對（`TemplateMatch`），需確認你的版本是否與官服圖片相同：

| 節點 | 模板圖片 | 說明 |
|---|---|---|
| `塔_偵測_buff選擇` | `爬塔_buff推荐图标1__...png` | Buff 選擇畫面的推薦圖示 |
| `塔_偵測_強化選卡` | 同上 | 強化選卡畫面的推薦圖示 |
| `塔_偵測_商店主界面` | `爬塔_商店主界面__...png` | 商店主界面特徵圖 |
| `塔_商店_關閉按鈕` | `星塔_格子_关闭__...png` | 商店詳情面板關閉按鈕 |
| `塔_商店_重置按鈕` | `爬塔_商店刷新按钮__...png` | 商店重置按鈕 |
| `塔_偵測_突發事件` | `爬塔_对话选项按钮__...png` | 突發事件選項藍色圓圈按鈕 |

若圖片不同，截取你版本的圖片放入 `assets/resource/YOUR_LANG/image/ClimbTower/`，
並在覆蓋 JSON 更新 `template` 路徑。

## Step 5：建立 Preset 作業檔案

Preset 是使用者自訂的 Buff 優先級清單，決定腳本在選卡時優先選哪些。

**檔案位置**：`agent/presets/你的作業名稱.json`

**格式**：

```json
{
    "config": {
        "skip_shop_rerolls": 1
    },
    "3": ["最高優先Buff名稱1", "最高優先Buff名稱2"],
    "2": ["次要Buff名稱1", "次要Buff名稱2"],
    "1": ["備選Buff名稱1"]
}
```

**config 選項說明**：

| 選項 | 預設值 | 說明 |
|---|---|---|
| `skip_shop_rerolls` | `0` | 跳過前幾次商店重置（0 = 每層都可重置） |

**選卡邏輯**：
- Priority 3 → 2 → 1 依序掃描，找到就選
- 全部不在清單內且有設定優先級 → 最多重置 2 次後選推薦圖示
- 沒有設定 preset → 直接選推薦圖示

---

## Step 6：新增 UI 選項

在 `assets/interface.json` 的 `爬塔作業` 節點的 `cases` 陣列加入新選項：

```json
{
    "name": "顯示名稱",
    "label": "顯示名稱",
    "pipeline_override": {
        "星塔_爬塔流程": {
            "action": {
                "type": "Custom",
                "param": {
                    "custom_action": "tower_loop",
                    "custom_action_param": "你的作業檔名.json"
                }
            }
        }
    }
}
```

## 購買策略說明

`tower_loop.py` 的商店購買邏輯（供參考，若有需要可 override）：

| 商品類型 | 判斷依據節點 | 購買條件 |
|---|---|---|
| Buff（潛能特飲等） | `塔_商店_buff類型` | 有折扣才買 |
| 已激活音符 | `塔_商店_音符類型` + `塔_商店_音符激活` | 直接買 |
| 未激活音符 | `塔_商店_音符激活` 未命中 | 跳過 |
| 其他 | 以上皆非 | 跳過 |

**商店重置**：掃完一輪後自動嘗試重置，無剩餘次數或前 N 輪（依 `skip_shop_rerolls`）則跳過。

## 常見問題

**Q：OCR 辨識到文字但點錯位置？**
調整對應節點的 `roi` 範圍，確保框住目標文字但不包含其他干擾文字。

**Q：購買確認按鈕點不到？**
可能是 OCR 模型輸出簡繁不同。在 `塔_偵測_確認按鈕` 的 `expected` 加入你版本的文字，
或使用 regex：`"確.?認"` 可同時匹配多種寫法。

**Q：強化按鈕偵測不到？**
`塔_偵測_強化可用` 的 `expected` 需包含你版本的「強化」與「免費」文字，
幣數符號（🪙）不必包含，OCR 通常不識別 emoji。

**Q：想新增 Preset 但不知道 Buff 叫什麼名字？**
進遊戲爬塔，log 裡會印出掃描到的所有 Buff 名稱（`scanning priority X: '名稱'`）。
